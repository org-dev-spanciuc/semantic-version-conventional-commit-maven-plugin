semantic-version-conventional-commit:add-git-hooks

Full name:

org-dev-spanciuc:semantic-version-conventional-commit-maven-plugin:1.0.0:add-git-hooks

Description:

A Mojo class to add git hooks to the project.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is not marked as thread-safe and thus does not support parallel builds.
  • Binds by default to the lifecycle phase: initialize.

Optional Parameters

Name Type Since Description
<commitMsgEnabled> boolean - Whether the hook is enabled.
Default value is: true.
User property is: commitMsgEnabled.
<commitMsgFileName> String - The hook's file name.
Default value is: commit-msg.
User property is: commitMsgFileName.
<commitMsgHeaderMaxLength> int - The max allowed length for commit header. For unrestricted length use:-1
Default value is: 150.
User property is: commitMsgHeaderMaxLength.
<commitMsgTypes> String - The list of comma separated allowed commit types.

Ex. "feat,fix,docs"


Default value is: feat,fix,chore,refactor,docs,style,test,perf,revert,merge.
User property is: commitMsgTypes.
<gitDirectory> String - The git directory name.
Default value is: .git.
User property is: gitDirectory.
<gitHooksDirectory> String - The git hook's directory name.
Default value is: .git/hooks.
User property is: gitHooksDirectory.
<gitHooksEnabled> boolean - Whether git hooks are enabled.
Default value is: true.
User property is: gitHooksEnabled.
